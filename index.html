Headline Patterns

Best immediate wins: developer platforms, productivity tools, support systems, and self-serve SaaS apps with clear REST/GraphQL docs.

Main blocker: not API complexity itself, but access gates: app review, developer-token approval, enterprise tenants, merchant accounts, paid plans, or partner programs.

Auth pattern: OAuth2 (47), API key / token (31), Unclear / account-based (7), API key (3). OAuth dominates collaboration/SaaS apps; API keys dominate developer, scraping, email, and document APIs; fintech and ads are more gated.

Easy wins: Twenty, Close, Copper, Freshdesk, Pylon, Gorgias, Telegram, Klaviyo, systeme.io, SendGrid, BigCommerce, Squarespace.

Needs outreach/account setup: DealCloud, WhatsApp Business, Google Ads, Meta Ads, LinkedIn Ads, Salesforce Commerce Cloud, Amazon Selling Partner, fanbasis, Waterfall.io, Clay, Snowflake, Paygent Connect.

Agent Workflow

1. Input
100 apps from assignment, category, app, website hint.
2. Evidence
Prefer official docs/API/auth pages. Avoid forum-only evidence.
3. Classify
Auth, self-serve/gated, API surface, MCP signal, verdict.
4. Confidence
High, Medium, Low based on docs clarity and fetchability.
5. Verify
20-app stratified manual check. Corrections shown honestly.

First-Pass Misses and Caveats

Misses / partials: Consensus, Otter AI.

Rule added after verification: apps with product pages but no clear public API docs are not marked ready, even if the company likely has internal APIs.

Known limitation: this agent does not create paid accounts, complete OAuth app reviews, or enter partner programs. Gated status is a finding, not a failure.
